# Tools

Once connected, your assistant can use the tools below to work inside Helix. You don't use them directly — you just ask for what you want, and the assistant picks the right tool for the job.

## Understand your project

- **inspect** — look at any object, actor, asset or level and read its details.
- **helix_docs** — read the official Helix documentation to answer "how do I…" questions.

## Build your world

- **control_actor** — add, move, rotate, duplicate or delete objects in the level.
- **manage_level** / **manage_level_structure** — work with levels and how they're organized.
- **build_environment** — set up environments and surroundings.
- **manage_geometry** — create and edit shapes and meshes.
- **manage_asset** — find, create and organize the things in your project.

## Control the editor

- **control_editor** — move the camera, play and stop, run commands, take screenshots.
- **system_control** — quality settings, performance checks and other editor tasks.
- **manage_tools** — turn individual MUSE tools on or off.

## Look and sound

- **manage_effect** — particles and visual effects.
- **manage_audio** — sounds and audio.
- **manage_sequence** — cinematics and sequences.
- **animation_physics** — animation and physics.

## Gameplay

- **manage_blueprint** — create and edit Blueprints (Helix's visual logic).
- **manage_character** — characters and the Character Creator.
- **manage_ai** — AI behavior.
- **manage_combat** — combat.
- **manage_gas** — abilities.
- **manage_interaction** — interactive objects.
- **manage_inventory** — inventories and items.
- **manage_networking** — multiplayer and networking.

_MUSE includes 200+ individual actions across these tools, and more are added over time._
